# Security Policy

# ===============

# 

# If you discover a security issue related to the binaries, build system, or

# release process, please report it responsibly.

# 

# Do not open a public GitHub issue for security‑sensitive information.

# 

# Instead, contact the repository maintainer directly using the email address

# listed in the project profile or GitHub settings.

# 

# When reporting a security issue, please include:

# 

# • A clear and concise description of the vulnerability  

# • Steps to reproduce the issue (if applicable)  

# • The affected platform, architecture, or binary  

# • Any potential impact or severity assessment (optional)

# 

# All security reports will be reviewed promptly, and you will receive a response

# as soon as possible.

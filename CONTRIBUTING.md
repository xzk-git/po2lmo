Contributing
============

Contributions are welcome, and there are several ways you can help improve this project:

• Reporting issues or bugs  
• Requesting support for additional architectures  
• Improving documentation or usage examples  
• Providing feedback on compatibility or performance across platforms  
• Submitting patches or enhancements to the build system or tooling  

If you would like to contribute code, please open an issue first to discuss your idea.
This helps ensure that your contribution aligns with the project’s goals and avoids
duplicate work.

Pull requests should be:

• Clear and focused  
• Well‑documented  
• Limited to a single purpose or change  
• Tested on at least one supported platform when possible  

All contributions—large or small—are appreciated.
